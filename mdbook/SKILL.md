# mdbook Workflows

## 1. Initialize Project

```bash
mdbook init my-book --title "Title" --ignore git
```

Standard structure:
```
my-book/
├── book.toml
├── .gitignore          # Must include: SUMMARY.md, book/
├── .github/workflows/mdbook.yml
├── src/
│   ├── README.md       # Book intro / index explanation
│   ├── SUMMARY.md      # Auto-generated by mdbook-summarizer (gitignored)
│   ├── _meta/          # Project metadata (prefixed with _)
│   │   ├── index.md    # Meta docs index
│   │   ├── list.yaml   # Entry list (source of truth)
│   │   └── list.md     # Human-readable entry list (auto-syncable)
│   ├── _tags/          # Auto-generated by mdbook-tagging
│   │   └── index.md    # Tag index table
│   └── {categories}/   # Content directories, organized by project convention
│       └── *.md        # Entry files
```

## 2. Entry List Management

`src/_meta/list.yaml` — the source of truth for all entries:

```yaml
title: Entry List
items:
  - name: "Entry Name"
    row: category_a      # Top-level grouping (determines directory)
    col: subcategory_a   # Sub-grouping (determines sub-directory)
    completeness: 100%
  - name: "Another Entry"
    row: category_b
    col: subcategory_b
    completeness: 80%
```

Fields:
- `name`: Display name
- `row`: Top-level category — determines the first directory level
- `col`: Sub-category — determines the second directory level
- `completeness`: Article completion percentage (e.g. `100%`, `65%`, `10%`)

The `row`/`col` scheme is flexible — adapt to the domain's natural grouping (e.g. alphabetical, by topic, by date).

### Add Entry Workflow

1. Append item to `list.yaml`
2. Create markdown file at `src/{row}/{col}/{name}.md`
3. Run `mdbook-summarizer --src src --auto-readme` to regenerate SUMMARY.md
4. Run `mdbook-tagging generate .` to update tag pages

### Entry File Template

```markdown
---
tags: [tag1, tag2, tag3]
---

# {Name}

## 概要

- **名称**: {name}
- **简介**: {description}

## 详情

{content}

## 参考资料

- {urls}
```

Adapt sections to the domain — the only requirement is the `tags:` frontmatter.

## 3. Tag System

### Frontmatter Tags

Every entry file uses YAML frontmatter:

```markdown
---
tags: [category, topic, subtopic]
---
```

### Generate Tag Pages

```bash
mdbook-tagging generate .
```

Scans all `tags:` in frontmatter, creates `src/_tags/{tag}.md` for each unique tag. Filenames: spaces replaced with hyphens (e.g. tag `Alice Japan` → `Alice-Japan.md`).

Each tag file is a backlink list:
```markdown
# Tag: topic

- [Entry A](../category_a/sub/EntryA.md)
- [Entry B](../category_b/sub/EntryB.md)
```

Note: relative paths from `_tags/` to entry dirs.

### Tag Index

`src/_tags/index.md` — a table listing all tags with article counts:
```markdown
# Tags

| Tag | 文章数 |
|-----|--------|
| [topic](topic.md) | 33 |
| [subtopic](subtopic.md) | 27 |
```

## 4. GitHub Actions Workflow

Bundled at `workflows/mdbook.yml` in this skill directory. Copy to the project:

```bash
mkdir -p .github/workflows
cp <skill_dir>/workflows/mdbook.yml .github/workflows/mdbook.yml
```

The workflow:
- Triggers on push to `main` + manual `workflow_dispatch`
- Installs mdbook, mdbook-summarizer, mdbook-tagging via cargo
- Runs `mdbook-tagging generate .` → `mdbook-summarizer --src src --auto-readme` → `mdbook build`
- Deploys to GitHub Pages via `actions/deploy-pages@v5`
- Uses concurrency group `"pages"` to avoid parallel deploys

## 5. Local Development

```bash
# Install tools
cargo install mdbook mdbook-summarizer mdbook-tagging

# Generate tags → src/_tags/
mdbook-tagging generate .

# Generate SUMMARY.md (auto-creates README.md for dirs missing one)
mdbook-summarizer --src src --auto-readme

# Build
mdbook build

# Serve locally
mdbook serve
```

## 6. book.toml Config

```toml
[book]
authors = ["Author"]
language = "zh"
src = "src"
title = "Book Title"

[output.html]
git-repository-url = "https://github.com/{owner}/{repo}"
edit-url-template = "https://github.com/{owner}/{repo}/edit/main/{path}"
```

## 7. .gitignore

```
SUMMARY.md
book/
```

SUMMARY.md is auto-generated — never commit it.

## 8. mdbook-summarizer

Rust CLI that generates `SUMMARY.md` from the source tree.

```bash
mdbook-summarizer --src src --auto-readme
```

| Flag | Description |
|------|-------------|
| `--src <DIR>` | Source directory (default: `src`) |
| `-o, --output <FILE>` | Output file (default: `<src>/SUMMARY.md`) |
| `--auto-readme` | Generate `README.md` for dirs without an index |
| `--dry-run` | Print without writing |
| `--check` | Exit error if SUMMARY.md is stale |

Behavior:
- `README.md` or `index.md` → directory entry
- Extracts first `# heading` as title
- Skips SUMMARY.md, dot dirs, temp files
- Sorts dirs before files, natural numeric order

## 9. mdbook-tagging

Rust CLI + mdbook preprocessor that generates tag index pages from frontmatter.

```bash
# Standalone
mdbook-tagging generate .

# As mdbook preprocessor (add to book.toml)
# [preprocessor.tagger]
# command = "mdbook-tagging preprocess"
```

Output: `src/_tags/{tag}.md` for each unique tag, plus `src/_tags/index.md` table.
